Introduction to RG Language

RG is a simple, C-style programming language designed for educational purposes. It combines the simplicity of basic programming constructs with powerful mathematical functions, making it ideal for learning programming concepts and solving computational problems.

Key features of RG language:

  • Simple, clean syntax inspired by C
  • Strong typing system with basic data types
  • Built-in mathematical functions
  • Support for arrays and basic data structures
  • Control flow statements for program logic

This manual provides a comprehensive guide to the RG language syntax, features, and usage.

Basic Syntax

RG programs consist of statements that end with semicolons. Comments, whitespace, and code blocks follow C-style conventions.

Comments

// This is a single-line comment

Statements

Each statement must end with a semicolon:

RG int x = 5;
RG_Print("Hello, world!");

Code Blocks

Code blocks are enclosed in curly braces:

if (condition) {
    // Code block
    statement1;
    statement2;
}

Variables and Data Types

All variable declarations in RG start with the RG keyword followed by the data type.

Basic Data Types

  • int - Integer values (e.g., 1, -5, 100)
  • float - Floating-point values (e.g., 3.14, -0.5, 2.0)
  • bool - Boolean values (true or false)

Variable Declaration and Initialization

// Declaration with initialization
RG int age = 25;
RG float pi = 3.14159;
RG bool isActive = true;

// Multiple variables of the same type
RG int x = 10, y = 20, z = 30;

Variable Naming Rules

  • Names must start with a letter or underscore
  • Names can contain letters, numbers, and underscores
  • Names are case-sensitive (age and Age are different variables)
  • Reserved keywords cannot be used as variable names

Control Flow

If-Else Statements

Conditional execution of code based on boolean expressions:

// Simple if statement
if (x > 10) {
    RG_Print("x is greater than 10");
}

// If-else statement
if (x > 10) {
    RG_Print("x is greater than 10");
} else {
    RG_Print("x is not greater than 10");
}

// If-else if-else statement
if (x > 10) {
    RG_Print("x is greater than 10");
} else if (x < 0) {
    RG_Print("x is negative");
} else {
    RG_Print("x is between 0 and 10");
}

For Loops

Execute code a specific number of times:

// Basic for loop
for (RG int i = 0; i < 5; i = i + 1) {
    RG_Print("Iteration:", i);
}

// Nested for loops
for (RG int i = 0; i < 3; i = i + 1) {
    for (RG int j = 0; j < 3; j = j + 1) {
        RG_Print("Position:", i, j);
    }
}

While Loops

Execute code as long as a condition is true:

// Basic while loop
RG int i = 0;
while (i < 5) {
    RG_Print("Iteration:", i);
    i = i + 1;
}

// While loop with complex condition
RG int sum = 0;
RG int num = 1;
while (sum < 100) {
    sum = sum + num;
    num = num + 1;
}
RG_Print("Final sum:", sum);

Break Statement

Exit a loop prematurely:

// Break out of a loop when a condition is met
RG int i = 0;
while (i < 10) {
    if (i == 5) {
        RG_Print("Breaking at i =", i);
        break;  // Exit the loop
    }
    RG_Print("Iteration:", i);
    i = i + 1;
}

Built-in Functions

RG provides several built-in functions for common operations:

Output Function

// Print to output
RG_Print("Hello, world!");
RG_Print("Value of x is:", x);
RG_Print("Multiple values:", x, y, z);

Mathematical Functions

sin(x) Sine of x (in radians) sin(3.14159 / 2) // Returns 1.0
cos(x) Cosine of x (in radians) cos(0) // Returns 1.0

Function Usage Examples

// Calculate the hypotenuse of a right triangle
RG float a = 3;
RG float b = 4;
RG float c = sqrt(pow(a, 2) + pow(b, 2));
RG_Print("Hypotenuse:", c);  // Outputs 5.0

// Calculate factorial
RG int n = 5;
RG int fact = factorial(n);
RG_Print("Factorial of", n, "is", fact);  // Outputs 120

Arrays

RG supports one-dimensional arrays for storing collections of values of the same type.

Array Declaration and Initialization

// Declare an array of integers with size 5
RG int numbers[5] = 0;  // Initialize all elements to 0

// Access and modify array elements
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
numbers[3] = 40;
numbers[4] = 50;

Array Operations

// Access array elements
RG int firstElement = numbers[0];
RG_Print("First element:", firstElement);

// Modify array elements
numbers[2] = numbers[0] + numbers[1];
RG_Print("Updated third element:", numbers[2]);

// Iterate through an array
for (RG int i = 0; i < 5; i = i + 1) {
    RG_Print("Element", i, ":", numbers[i]);
}

Array Manipulation Functions

// Insert a value into an array
insert(numbers, 100);   // Adds 100 to the array

// Delete an element at a specific index
delete(numbers, 2);     // Removes the element at index 2

Complete Examples

Example 1: Calculate Average

// Calculate the average of an array of numbers
RG int numbers[5] = 0;
RG int sum = 0;

// Initialize array with values
numbers[0] = 10;
numbers[1] = 20;
numbers[2] = 30;
numbers[3] = 40;
numbers[4] = 50;

// Calculate sum
for (RG int i = 0; i < 5; i = i + 1) {
    sum = sum + numbers[i];
}

// Calculate average
RG float average = sum / 5;
RG_Print("Average:", average);

Example 2: Fibonacci Sequence

// Generate Fibonacci sequence up to n terms
RG int n = 10;
RG int fib[10] = 0;

// Initialize first two terms
fib[0] = 0;
fib[1] = 1;

// Generate the rest of the sequence
for (RG int i = 2; i < n; i = i + 1) {
    fib[i] = fib[i-1] + fib[i-2];
}

// Print the sequence
RG_Print("Fibonacci Sequence:");
for (RG int i = 0; i < n; i = i + 1) {
    RG_Print("Term", i, ":", fib[i]);
}

Example 3: Prime Number Checker

// Check if a number is prime
RG int num = 17;
RG bool isPrime = true;

if (num <= 1) {
    isPrime = false;
} else {
    for (RG int i = 2; i < num; i = i + 1) {
        if (num % i == 0) {
            isPrime = false;
            break;
        }
    }
}

if (isPrime) {
    RG_Print(num, "is a prime number");
} else {
    RG_Print(num, "is not a prime number");
}

Using the RG Compiler

  • Code Editor: Where you write your RG code
  • Output Panel: Displays program output, errors, and warnings
  • Token Panel: Shows the lexical tokens generated from your code
  • Assembly Panel: Displays the generated assembly code
  • Symbol Table: Lists all variables and their properties

Compilation Process

  1. Write Code: Enter your RG code in the editor
  2. Run: Click the "Run" button to compile and execute your code
  3. View Results: Check the output panel for program results
  4. Debug: If there are errors, check the errors panel for details

Compiler Features

  • Syntax Highlighting: Code is colored for better readability
  • Error Reporting: Detailed error messages help identify issues
  • Line Numbers: Help locate code positions quickly
  • Token Analysis: View the lexical structure of your code
  • Assembly Generation: See how your code translates to assembly

Compiler Architecture

The RG Compiler is a true compiler that transforms source code into an executable intermediate representation through several phases:

1. Lexical Analysis (Lexer)

The lexer breaks down the source code into tokens such as keywords, identifiers, literals, and operators. This is the first step in understanding the structure of the program.

// Source: RG int x = 5;
// Tokens: [RG, int, x, =, 5, ;]
2. Syntax Analysis (Parser)

The parser analyzes the tokens according to the grammar rules of the RG language and builds an Abstract Syntax Tree (AST) that represents the hierarchical structure of the program.

// AST for x = 5 + 3;
// AssignmentExpression {
//   name: "x",
//   value: BinaryExpression {
//     left: 5,
//     operator: "+",
//     right: 3
//   }
// }
3. Semantic Analysis

This phase checks for semantic errors such as undeclared variables, type mismatches, and invalid operations. It also builds a symbol table to track variables and their properties.

RG Compiler: Detailed Analysis

The RG Compiler project is a comprehensive implementation of a compiler for the RG programming language. Let's analyze the key components and features that make this a true compiler:

1. Compiler Architecture

The project follows the classical compiler architecture with distinct phases:

Lexical Analysis (js/lexer.js)
  • Implements a tokenizer that breaks source code into tokens
  • Recognizes keywords, identifiers, literals, operators, and punctuation
  • Handles token position tracking for error reporting
  • Example: RG int x = 5; is tokenized into [RG, int, x, =, 5, ;]
Syntax Analysis (js/parser.js)
  • Implements a recursive descent parser
  • Constructs an Abstract Syntax Tree (AST) from tokens
  • Handles grammar rules for expressions, statements, and declarations
  • Performs syntax error detection and reporting
  • Example: Parses x = 5 + 3; into an AST with assignment and binary expression nodes
Semantic Analysis (js/semantic.js)
  • Performs type checking and validation
  • Builds and maintains a symbol table for variables and their properties
  • Detects semantic errors like undefined variables or type mismatches
  • Generates assembly-like intermediate code
  • Example: Validates that variables are declared before use and operations have compatible types
Code Generation (js/codegen.js)
  • Transforms the AST into target code
  • Implements code generation for all language constructs
  • Handles variable declarations, expressions, control flow, and I/O
  • Example: Generates code for loops, conditionals, and mathematical operations

2. Intermediate Code Generation

The compiler generates assembly-like intermediate code:

; Declare variable x
MOV R0, 10
STORE [x], R0

; Declare variable y
MOV R1, 5
STORE [y], R1

; Calculate x + y
LOAD R2, [x]
LOAD R3, [y]
ADD R4, R2, R3
STORE [sum], R4

This intermediate representation is a key characteristic of compilers, serving as a bridge between the high-level source code and the execution environment.

3. Symbol Table Management

The compiler maintains a sophisticated symbol table:

  • Tracks variable names, types, and values
  • Handles different data types (int, float, bool, string)
  • Supports arrays with proper bounds checking
  • Prevents variable redeclaration
  • Ensures type compatibility in operations

Example from the code:

this.symbolTable[varName] = {
    type: node.dataType || 'int',
    value: null
};

4. Type System

The compiler implements a strong type system:

  • Supports basic types: int, float, bool, and string
  • Performs type checking for operations
  • Handles type conversions where appropriate
  • Validates array operations and bounds
  • Reports type-related errors with detailed messages

5. Error Handling

The compiler provides comprehensive error detection and reporting:

  • Syntax errors with position information
  • Semantic errors with detailed explanations
  • Runtime error detection (e.g., division by zero)
  • Helpful suggestions for fixing common errors
  • Example error: "Variable 'x' already declared" or "Undefined variable 'y'"

6. Control Flow Analysis

The compiler properly handles control flow constructs:

  • If-else statements with condition evaluation
  • While and for loops with proper scoping
  • Break statements with validation
  • Nested control structures
  • Label generation for jumps in the intermediate code

7. Optimization Techniques

While basic, the compiler does implement some optimizations:

  • Constant folding for literal expressions
  • Dead code elimination in conditional branches
  • Efficient register allocation in the intermediate code

8. File Structure and Organization

The project is well-organized into modular components:

  • js/lexer.js: Tokenization and lexical analysis
  • js/parser.js: Syntax analysis and AST construction
  • js/semantic.js: Semantic analysis and validation
  • js/codegen.js: Code generation from AST
  • js/compiler.js: Main orchestration and UI integration
  • index.html: User interface for the compiler
  • css/: Styling for the compiler interface

9. Execution Model

The compiler follows a complete compilation process:

  1. Source code is fully analyzed before execution begins
  2. The entire program is transformed into an intermediate representation
  3. The intermediate code is then executed in a simulated environment
  4. Results are displayed after the compilation and execution process

Conclusion

The RG Compiler project is a legitimate compiler that transforms source code in the RG language into an executable intermediate representation. It implements all the essential phases of compilation: lexical analysis, syntax analysis, semantic analysis, and code generation. While it includes an integrated execution environment rather than generating standalone executables, this is a common approach for educational compilers and doesn't diminish its status as a true compiler.

The project demonstrates a deep understanding of compiler construction principles and provides a valuable tool for learning about programming language implementation.
